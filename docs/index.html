Claude Code plugin

AI detection and humanization for Claude Code.

Two skills backed by the Rephrasy API: /rephrasy:ai-detect scores any text 0 (human) to 100 (AI) with a per-sentence breakdown, and /rephrasy:humanize rewrites AI-generated text to read as human. Defaults to Rephrasy's v3 model with professional style.

Quick start

Add the marketplace, install the plugin, reload. Requires Claude Code with plugin support, python3 (the scripts are stdlib-only, zero pip dependencies), and a REPHRASY_API_KEY env var from rephrasy.ai (paid API).

# Install
/plugin marketplace add jhubbardsf/rephrasy-skills
/plugin install rephrasy@rephrasy-skills
/reload-plugins

# Invoke
/rephrasy:ai-detect <file>
/rephrasy:humanize <file>
# One-line installer
curl -fsSL https://raw.githubusercontent.com/jhubbardsf/rephrasy-skills/main/install.sh | bash

# Requires
# Claude Code with plugin support
# python3 (stdlib-only, zero pip dependencies)
# REPHRASY_API_KEY from rephrasy.ai (paid API)

Built against the live API, not the docs

Detection that shows its work, humanization with control, and verdicts calibrated from real API responses.

Per-sentence detection

Depth-mode scoring shows exactly which sentences read as AI, so rewrites target the right lines.

Humanize with control

Rephrasy v3 model with professional style by default; creative/journalistic/no-style and other models one flag away.

Calibrated verdicts

Score bands calibrated against live API data, including the fact that formal human prose scores 60–75 on this detector (a verified-human sample scored 70.28; casual human text 7.35).

Iterate until it passes

Built-in loop workflow: humanize → score → repeat (max 3 passes), keeps the best-scoring version, never burns paid passes on API outages thanks to a strict exit-code contract (0 ok / 1 over threshold / 2 error).

Zero dependencies

Stdlib-only Python helpers; no pip install, no node_modules. Atomic file writes, clean one-line errors, costs surfaced after every call.

Docs-vs-reality tested

Behavior encoded from live API verification, not the marketing docs (e.g. score direction is 0=human/100=AI in every mode, despite docs claiming otherwise; invalid keys actually return HTTP 400).

Usage examples

Point either skill at a file or paste prose inline. Humanized output lands next to the input.

/rephrasy:ai-detect draft.md
/rephrasy:ai-detect "paste some prose here"
/rephrasy:humanize draft.md            # -> draft_rephrasy.md
/rephrasy:humanize draft.md --verify   # re-score the result
/rephrasy:humanize draft.md --iterate --threshold 40
/rephrasy:humanize draft.md --style creative
